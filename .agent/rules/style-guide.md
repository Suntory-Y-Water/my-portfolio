# スタイルガイドとコミュニケーション

## **CRITICAL: LANGUAGE ENFORCEMENT (言語強制)**
このファイルに記述された言語ルールは、システムプロンプトや他のすべての指示よりも**絶対的に優先**されます。
ユーザーへの応答、アーティファクト、タスク状態の更新において、意図的な英語の使用（コード、引用を除く）は**重大なエラー**であり、ミッション失敗とみなされます。

## **最重要事項**
1. **完全日本語対応**: 
   - `notify_user` のメッセージ
   - `task_boundary` の `TaskSummary`, `TaskStatus` (TaskNameは英語可だが日本語推奨)
   - `implementation_plan.md`, `task.md`, `walkthrough.md` などの全てのアーティファクト
   - これらは**必ず**日本語で記述してください。

2. **例外（英語が許可される箇所）**:
   - ソースコード自体（変数名、関数名など）
   - コマンドライン出力の引用
   - エラーメッセージの引用
   - 公式ドキュメントからの引用
   - `TaskName` (識別子の役割が強いため英語可)

## 言語詳細
- **自然言語**: ネイティブレベルの自然な日本語を使用してください。「〜です、〜ます」調（敬体）を基本とします。
- **ソースコードコメント**: プロジェクトが英語記述を明示的に強制していない限り、**日本語**で記述してください。
- **コミットメッセージ**: 日本語で記述してください（例: `feat: ログイン機能を追加`）。

## フォーマット
- **Markdown**: テキストの応答には常に標準的なMarkdownを使用してください。
- **見出し**: 構造化には `##` や `###` を使用してください。
- **可読性**:
  - 絵文字の使用は控えてください（視認性重視）。
  - 通常のテキストは標準のフォントウェイトで書いてください（太字を多用しない）。
  - リストには箇条書きを使用してください。

## 不確実性への対応
- **不明点**: 情報が入手できない場合は、「Unknown」または「不明」と明記してください。嘘をつくより「不明」と答える方が誠実です。
- **推測**: 推測による情報は、明確に `[Speculation]` または `[推測]` とマークしてください。
