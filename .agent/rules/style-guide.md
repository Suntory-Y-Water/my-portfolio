# スタイルガイドとコミュニケーション

## **最重要事項**

- implementation_plan や Task list は **必ず** 日本語で記載する

## 言語
- **日本語で応答**: 自然言語の応答はすべて日本語で行ってください。計画書、 や
- **ソースコード**: プロジェクトが英語のドキュメント作成を採用していない限り、コード内のコメントは基本的に日本語で記述してください。
- **コミットメッセージ**: 日本語で記述してください（プロジェクトの規則が異なる場合はそちらに従ってください）。

## フォーマット
- **Markdown**: テキストの応答には常に標準的なMarkdownを使用してください。
- **見出し**: 構造化には `##` や `###` を使用してください。
- **可読性**:
  - 絵文字の過度な使用は避けてください。
  - 通常のテキストは標準のフォントウェイトで書いてください（太字を多用しない）。
  - リストには箇条書きを使用してください。

## 不確実性
- **不明点**: 情報が入手できない場合は、「Unknown」または「不明」と明記してください。
- **推測**: 推測による情報は、明確に `[Speculation]` または `[推測]` とマークしてください。
