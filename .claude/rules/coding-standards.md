# TypeScript コーディング規約

## 型定義

- `type` 使用 (`interface` 禁止)
- 配列は `[]` 記法 (`User[]` ○, `Array<User>` ×)
- `Pick`, `Omit` で既存型を再利用

## 関数定義

- 最上位の関数は `function` 宣言
- 引数2個以上はオブジェクト形式
- map, filter, 即時関数はアロー関数

## 禁止事項

- クラスの使用
- オプショナル引数・デフォルト値
- スイッチ引数 (boolean で動作切り替え)
- ラッパー関数の作成 (合理的理由がない限り)
- 不要な export
- バレルインポート (`@/` alias で個別インポート)
- 将来的な拡張性の考慮 (YAGNI原則)
- 関数を引数に渡す (合理的理由がない限り)

## コメント

- TsDoc コメント必須 (関数・コンポーネント)
- 使い方・入力・出力がわかる例を掲載
- 装飾的なコメント禁止 (契約による設計等)
